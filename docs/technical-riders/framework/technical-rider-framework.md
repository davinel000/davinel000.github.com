# Technical Rider Framework

This is the unified structure for all future technical riders. It is meant for installation-based artworks and can be shortened or expanded depending on the project.

## 1. Header

- title of artwork
- year
- one representative photo
- artist name
- contact email / phone
- version number of rider
- date of last update

## 2. Short Description

Keep this to 60-120 words.

It should explain:

- what the work is
- what visitors experience
- why specific technical conditions matter

## 3. Core Components

List only the essential artistic and technical components.

## 4. Dimensions / Duration / Scalability

- installation footprint
- ideal room size
- packed dimensions and weight
- duration of loops or cycles
- continuous operation or timed activations
- whether the work can be scaled down or adapted

## 5. Spatial Requirements

- darkness / ambient light level
- wall or floor requirements
- acoustic conditions
- humidity / dust / temperature limits
- visitor circulation
- whether a separate room is preferred or required

## 6. Technical Requirements Provided by Venue

- power
- blackout materials
- lighting fixtures
- mounting hardware
- plinths / stands / barriers
- service monitor or furniture if needed
- staff support during setup

## 7. Components Provided by Artist

- devices
- stands
- controllers
- cables and adapters
- consumables
- packaging units

## 8. Sound

- whether the work has sound
- required volume range
- whether sound bleed is acceptable
- whether headphones are allowed or not

## 9. Light

- whether full blackout is required
- whether colored safety light is acceptable
- which light sources are part of the work
- which extra venue lights are needed

## 10. Safety

- laser safety
- flashing-light warning
- dark-room navigation
- hot surfaces / ventilation
- cable and trip hazards
- visitor distance from equipment
- whether mediation or supervision is recommended

## 11. Transport

- number of packages
- dimensions and weight of each
- fragility notes
- special handling instructions

## 12. Setup / Deinstallation

- setup time
- deinstallation time
- what must be done before artist arrival
- what requires artist presence
- how many assistants are needed

## 13. Daily Operation

- startup sequence
- shutdown sequence
- daily checks
- cleaning
- reboot instructions
- who is allowed to do these tasks

## 14. Maintenance

- expected maintenance frequency
- known weak points
- spare parts available or not
- remote support possible or not
- when artist intervention is required
- how service travel should be handled

## 15. Insurance Value

- itemized value if useful
- total insurance value
- note if value changes depending on version of work

## 16. Optional Curatorial Notes

- preferred viewing conditions
- seating recommendations
- ideal atmosphere
- neighboring works to avoid

## 17. Appendix

- floor plan
- power diagram
- setup photos
- warning signage
- startup checklist

## Design Rules

- keep it to 3-5 pages for most works
- one visual identity across all riders
- same typefaces, same grid, same section order
- use short paragraphs and flat bullet lists
- avoid mixing conceptual text and logistics inside the same section
- keep one clean first page with image, short intro, and key facts
- keep one final page for setup, maintenance, and insurance
